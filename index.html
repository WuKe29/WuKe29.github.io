Gait Recognition with IMU Data & Explainable AI Feb 2026 – Apr 2026

📌 Problem: Academic research project aiming to recognize individuals and walking speeds using wearable inertial measurement unit (IMU) data. The challenge lies in high-dimensional time-series sensor data and the need for model interpretability.

📊 Data: Kuopio Gait Dataset (University of Eastern Finland) – IMU extracted data from 51 healthy participants walking at slow, comfortable, and fast speeds. Used 7 IMU sensors (pelvis, thighs, shanks, feet). Preprocessed and integrated raw signals to address missing values and synchronization issues.

⚙️ Approach: Extracted 522 hand-crafted features (time/frequency domain) and a 42-feature baseline. Applied Random Forest, CNN, and traditional ML models. Used SHAP and Gini impurity for variable importance analysis, sensor importance ranking, and optimal sensor combination. Compared full 522-dim features vs. dimensionality reduction (48-dim after feature selection) vs. end-to-end deep learning.

🏆 Outcome: Baseline (42 features) achieved 96% accuracy. Full 522 features, CNN, and reduced 48 features all achieved >99.5% accuracy. Identified optimal sensor set (pelvis + shanks) reducing hardware cost while maintaining high accuracy. Provided model interpretability insights for gait analysis under different speeds.

🤝 My contribution: Team leader. Designed all experiments, co-wrote code for preprocessing, feature extraction, model training (Random Forest, CNN), SHAP analysis, and sensor combination search. Reviewed and integrated team contributions.

🏆 Outcome: Delivered actionable insights: identified high-value user segments, evaluated marketing channel ROI, and provided recommendations to improve user engagement.
